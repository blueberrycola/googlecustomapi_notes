Managing your search engine:

Management is done via the control panel
You can create more than one search engine under one google account
Your search engine page contains the following components:

  • ctrl panel: administrative console
  • Statistics: dashboard that displays the number of queries processed by your engine. If enough queries of the same type are googled it can be seen as a popular term
  • delete: permanently delete your search engine

Control Panel: Basics

lets you define public info and general settings about your engine.
{name, desc, keywords, edition}
name: name of engine
desc: description of engine
keywords is optional: used to boost certain webpages in your results. 100 char limit List words or phrases. NOTE KEYWORDS DO NOT DEMOTE OR FILTER THAT DONT CONTAIN KEYWORDS. used for fine tuning. Usually the last step
edition: Free with ads or Nonprofit
searchEngine ID: the id of your engine aka cx
public url: a link to the google hosted page of your search engine
safesearch: enable/disable safesearch
region: customize based on selected region
language: language of the interface of the search engine
sites to search: the sites and its pages included via url

http://www.example.com -> displays homepage only
.example.com/ -> displays all pages in the given domain

Control Panel: Refinements

Refinements are a way for you to categorize sites by topics (or perhaps subdirectories of one site?)

You can create refinement labels that you associate with the sites you listed in the sites tab.
To create a refinement label, click Add Refinement and define the settings. One site can have more than one label

Control Panel: Promotions

Promotions let you show certain sites at the top page. Labels search result as a promotion for involuntary consumer. 🤑

Control Panel: Autocompletions

Offers query options to your users as they enter the search queries on your search box.

Control Panel: Advanced + Admin

When you’ve outgrown the control panel and want to start tinkering with the advanced features, you should consider using the Programmable Search Engine context and annotations files. The context file is in XML format, while the annotations file can be in OPML, TSV, or Programmable Search XML format.

An administrative account will allow collaborators.
