Overview

A context file describes the structure of your search engine and defines its behavior.

Example: describes a search engine for climate change, img search enabled.

<CustomSearchEngine keywords="climate &quot;global warming&quot; &quot;greenhouse gases&quot;" language="en" encoding="UTF-8"> <Title>RealClimate</Title> <Description>Science behind global warming and climate change.</Description> <Context> <BackgroundLabels> <Label name="_cse_hwbuiarvsbo" mode="FILTER"/> <Label name="_cse_exclude_hwbuiarvsbo" mode="ELIMINATE"/> </BackgroundLabels> </Context> <LookAndFeel nonprofit="false"/> <ImageSearchSettings enable="true" layout="1" /> </CustomSearchEngine>

Elements

CustomSearchEngine: Encloses all information about the search engine, contains the following attributes:

  • keywords: specifies a list of space seperated keywords

  • language: user interface language

  • encoding: Specifies the encoding for your search results page

  • autocompletions: indicates wheater should generate glossary autocompletions for your search engine. Allowed values are true or false.
    Title: The name of your engine
    Description: Description of your engine
    Context: hols information about labels. Includes the child elements BackgroundLabels and Facet
    BackgroundLabels: Container for labels used to promote include or exclude pages
    Label: Used to promote sites or determine inclusion/exclusion of sites. Has the following attributes

  • name: label name

  • mode FILTER or ELIMINATE: (promotes or exclusively shows preferred sites) (excludes site)
    LookAndFeel: Determines whether your search results show ads, etc
    AdSense: Associates the search engine with your AdSense account.
    ImageSearchSettings: Specifies if img search enabled. Has the following attributes:
    enable: specifies whether to enable
    layout: specifies layout

  • Value

    Layout

    0

    1

    2 (default)
